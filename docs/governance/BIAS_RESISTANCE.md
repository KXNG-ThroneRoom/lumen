# Lumen Bias Resistance

Version: v0.2-local-deterministic

## Position

Lumen cannot guarantee perfect neutrality. No human-built system can.

Bias resistance in Lumen comes from transparent process, contestable methods, visible uncertainty, auditability, and forkability.

## Bias Risks

Known risks include:

- source selection bias;
- language and geography bias;
- institutional bias;
- anti-institutional bias;
- popularity bias;
- recency bias;
- availability bias;
- automation bias;
- interface-induced authority;
- false balance;
- hidden ideological weighting;
- contributor capture;
- benchmark overfitting;
- incomplete source metadata.

## Process Controls

Lumen uses these controls:

- public governance rules;
- documented scoring methodology;
- visible confidence reasons;
- source-independence tracking;
- anonymous-source caps;
- primary-evidence preference with provenance checks;
- limitation disclosures;
- adversarial review;
- audit logs;
- deterministic fixtures for MVP testing;
- challenge and correction process;
- forkability.

## Equal Scrutiny Rule

Lumen applies the same method to all actors.

The system must not assume that any source class is automatically correct or incorrect, including mainstream media, alternative media, governments, corporations, NGOs, activists, academics, experts, anonymous officials, independent creators, or opposition groups.

Evidence and provenance govern confidence. Identity alone does not.

## Source Diversity Without Source Laundering

Diverse sources can improve analysis only when they contribute independent evidence or perspective.

Lumen must not treat many copies of the same origin as independent corroboration.

Source-spread analysis should distinguish:

- independent reporting;
- shared wire source;
- press-release repetition;
- circular citation;
- same anonymous briefing;
- common ownership or funding;
- repeated language;
- geographic or linguistic concentration.

## Narrative Analysis Safeguards

Narrative analysis must be labeled as analysis, not fact.

A detected frame does not prove deception. It indicates a pattern users may inspect.

Narrative modules should expose:

- detected frame;
- textual basis;
- uncertainty;
- limitation;
- audit reference.

## Interface Safeguards

The interface must avoid certainty theater.

Required safeguards:

- show confidence reasons near confidence bands;
- show unresolved questions in briefings;
- show limitations near affected outputs;
- label stubs and fixtures clearly;
- expose audit references;
- avoid verdict-style simplification without context;
- avoid hiding contradictions.

## Adversarial Review

Bias resistance requires challenge.

Reviewers should test whether Lumen handles politically opposed, institutionally opposed, and narrative-opposed cases under the same rules.

Challenges should be documented with:

- challenged output;
- alleged bias type;
- evidence for the challenge;
- rule implicated;
- proposed remedy;
- resolution;
- audit or issue reference.

## MVP Boundary

v0.2 does not claim comprehensive neutrality, live coverage, global source balance, or production-grade analysis. It demonstrates transparent local primitives that can be inspected and challenged.
